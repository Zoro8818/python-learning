# Python 学习资料整理规则

## 目标

这个仓库用于整理 Python 学习代码、学习笔记和项目作品，方便以后上传 GitHub 展示。

核心原则：

- `D:\python-project` 是对外展示仓库，尽量保持干净、清楚、可阅读。
- `D:\workspace\38-python-b-python-ai-python\outputs` 是内部学习记录区，用来保存每日学习日记、计划、模板和课堂小结。
- 不是所有学习过程都要上传 GitHub；只有整理过、能给别人看的内容才放进 `D:\python-project`。

## 两个位置的分工

### 1. GitHub 展示仓库

位置：

```text
D:\python-project
```

用途：

- 保存 Python 学习代码。
- 保存整理后的学习笔记。
- 保存可以展示的项目作品。
- 作为以后求职作品集的一部分。

不建议放：

- 每日流水账日记。
- 临时草稿。
- API Key、Token、账号密码、Cookie、支付信息。
- `.venv`、`.idea`、`__pycache__` 等环境和编辑器文件。

### 2. 内部学习记录区

位置：

```text
D:\workspace\38-python-b-python-ai-python\outputs
```

用途：

- 保存每日学习日记。
- 保存 Python 就业路线计划。
- 保存刷题模板。
- 保存课堂小结和课堂练习。
- 保存 AI 陪练过程中的复盘记录。

这个位置不要求直接上传 GitHub。

## 命名规则

### Python 代码文件

统一格式：

```text
编号_主题.py
```

示例：

```text
01_字面量与变量.py
12_数据容器_list.py
13_列表基础.py
14_字典基础.py
```

规则：

- 用两位数字编号，方便排序。
- 主题尽量短，能看懂就行。
- 中文和英文混合时，用 `_` 分开。
- 不要在文件名前加 `├──`、`│`、空格树形符号。

### 学习笔记文件

统一格式：

```text
编号_主题.md
```

示例：

```text
01_数据容器.md
02_列表常用方法.md
03_字典基础.md
```

### 课堂小结文件

课堂小结文字版固定放到：

```text
学习笔记/课堂小结/
```

课堂小结图片固定放到：

```text
学习笔记/images/课堂小结/
```

### 项目目录

统一格式：

```text
Project编号_项目名
```

示例：

```text
Project002_Excel工资统计
Project003_多Excel合并工具
```

## README 规则

每个主要目录都保留一个 `README.md`。

README 至少写清楚三件事：

```text
1. 这个目录是学什么的
2. 当前有哪些文件
3. 推荐学习顺序
```

项目类 README 还要写清楚：

```text
1. 项目目标
2. 输入文件
3. 输出文件
4. 运行方法
5. 当前已完成功能
6. 后续升级方向
```

## 图片规则

学习截图不要直接堆在 `学习笔记` 根目录。

普通学习图片推荐放到：

```text
学习笔记/images/
```

课堂小结配套图片固定放到：

```text
学习笔记/images/课堂小结/
```

课堂小结文字版 Markdown 固定放到：

```text
学习笔记/课堂小结/
```

图片命名建议：

```text
数据容器类型.png
列表的定义及访问.png
第16节_字符串格式化小结.png
```

## 上传 GitHub 前检查

上传前先检查：

```text
1. 是否包含 .venv、.idea、__pycache__
2. 是否包含账号、密码、Token、API Key、Cookie
3. 是否包含付款截图、订单号、隐私资料
4. README 是否能说明这个仓库是做什么的
5. 文件名是否干净、可读、好排序
```

## 当前执行原则

现阶段不要一次性大搬家。

优先顺序：

```text
1. 先统一命名规则
2. 再整理 README
3. 再移动图片到 images
4. 最后把成熟项目整理进 项目实战
```

每次只整理一小块，整理完再上传或提交，避免一次改太多导致回退困难。
