# 项目实战

本目录用于保存可以逐步打磨成作品集的 Python 项目。

这里不是放零散练习题的地方，而是放“能解决一个具体问题”的小工具或完整项目。

## 项目收录标准

一个项目放进本目录前，至少要说清楚：

```text
1. 项目目标：解决什么问题
2. 输入：需要什么文件或数据
3. 处理：Python 做了什么
4. 输出：生成什么结果
5. 运行方法：怎么运行
6. 后续升级：还能怎么改进
```

## 推荐项目结构

每个项目单独一个文件夹，推荐格式：

```text
Project编号_项目名/
  README.md
  src/
  input/
  output/
  docs/
  tests/
```

目录用途：

- `README.md`：项目说明，给自己和别人看。
- `src/`：Python 源代码。
- `input/`：示例输入文件。
- `output/`：示例输出结果。
- `docs/`：项目笔记、设计思路、截图说明。
- `tests/`：测试脚本或验证样例。

## 不建议放入本目录的内容

- 单个语法练习文件。
- 每日学习日记。
- 临时测试代码。
- `.venv`、`.idea`、`__pycache__`。
- API Key、Token、账号密码、Cookie、支付信息等敏感内容。

## 项目完成标准

一个项目算“可以展示”，至少要满足：

```text
1. README 能讲清楚项目目标和运行方法
2. 有 src 源代码
3. 有 input 示例数据
4. 有 output 示例结果
5. 运行后能稳定得到结果
6. 不包含敏感信息
```
